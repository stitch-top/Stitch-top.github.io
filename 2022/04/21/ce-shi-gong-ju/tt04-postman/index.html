Postman

Postman是一个简单的接口测试工具，被广泛应用于API设计、API构建、API测试等工作环节，也同样适用于API安全测试和管理。

1 Postman

  • Postman
    • 官方下载，安装后注册账号，登录使用：https://www.postman.com/downloads/。
    • Create new collection：添加用例集，Documentation是用例集描述(非必填项)。
    • Add folder：添加模块。
    • Add request：添加请求，Headers(请求头)、Body(请求体，为Json时选择raw)。
    • Export：导出用例集。
    • Import：导入用例集，若已存在同名的用例集，则Import as Copy，即导出一个拷贝份，不建议Replace替换。

2 常用断言

  • 常用断言
    • 借助JavaScript语言进行代码的编写，自动判断预期结果与实际结果是否一致。
    • 断言代码写在请求的Tests标签中，请求Response中的Test Results即断言结果。
    • 工作原理(由上往下依次执行)
      • Postman界面：组织http请求方法、请求头、请求体、url等。
      • Postman内部：分析http请求方法、请求头、请求体、url等。
      • 服务器：接收Postman发送过来的请求后，解析http请求，回发http响应。
      • Postman内部：执行断言代码，得到断言结果。
      • Postman界面：在Test Results中显示断言结果。

3 变量设置

  • 变量设置
    • 全局变量：在Postman中全局生效的变量，全局唯一。
      • 代码设置：pm.globals.set("var_name", value);，var_name是全局变量名，value是全局变量值。
      • 代码获取：var value = pm.globals.get("var_name");，var_name全局变量名，value接收值变量。
      • 请求参数获取(即Postman界面获取)：{{ var_name }}，var_name是全局变量名。
    • 环境变量：特定环境(生产环境、开发环境、测试环境)下生效的变量，在本环境内唯一。
      • 代码设置：pm.environment.set("var_name", value);，var_name是环境变量名，value是环境变量值。
      • 代码获取：var value = pm.environment.get("var_name");，var_name环境变量名，value接收值变量。
      • 请求参数获取(即Postman界面获取)：{{ var_name }}，var_name是环境变量名。
